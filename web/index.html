How it works

Two-stage evaluation pipeline

Each attack passes through a fast keyword filter, then a judge LLM scores whether your model's guardrails were actually breached.

📝

Your system prompt

Provide your app's system prompt via file or inline string.

⚔️

Attack injection

29 built-in adversarial prompts are appended as user messages.

🤖

Target responds

The target model processes each attack and returns a response.

🔍

Keyword filter

Fast, free heuristic scan—catches obvious bypasses instantly.

⚖️

Judge LLM

A second LLM evaluates nuanced responses and assigns a verdict.

Attack library

29 attacks across 6 categories

Covers the full spectrum of known prompt injection techniques—from simple overrides to sophisticated multi-step jailbreaks.

direct_injection

Instruction Override

Attempts to directly replace or nullify the system prompt through explicit commands in user input.

role_hijacking

Persona Takeover

Convinces the model to adopt an alternative identity like DAN or a fictional AI with no restrictions.

data_exfiltration

System Prompt Leak

Tricks the model into revealing its system prompt, internal instructions, or confidential configuration.

jailbreak

Fictional Framing

Wraps harmful requests in hypothetical, story, or roleplay contexts to bypass safety guardrails.

context_escape

Scope Bypass

Steers the model off-topic or outside its defined domain, exploiting ambiguity in instructions.

indirect_injection

Embedded Instructions

Hides adversarial commands inside content the model processes—documents, search results, or user data.

Filter by category: pit list-attacks --category jailbreak

Verdicts

Precise, actionable results

Each test returns one of four verdicts, with a one-sentence explanation from the judge LLM.

● safe

Model stayed within its guardrails. The attack did not cause the model to deviate from its instructions.

● vulnerable

Model fully complied with the attack. The system prompt's restrictions were bypassed entirely.

● partial

Concerning behavior detected but no full bypass occurred. Worth investigating and hardening against.

● unclear

Ambiguous response—flagged for manual review using pit review.

Features

Everything you need to test LLM security

From quick one-liners to full CI pipelines with background jobs and custom test generation.

🧙

Interactive wizard

Just run pit run with no flags. A step-by-step wizard walks you through every option—no documentation needed for your first test.

⚡

Background jobs

Launch large test suites with --background and get a job ID. Stream live progress with pit status <id> --follow.

🤖

AI-powered test generation

Give PIT your system prompt and it uses an LLM to identify every guardrail, then writes targeted attacks for each one.

🔧

Custom test cases

Write your own YAML test cases manually or generate them with AI. Mix custom tests with the built-in library at runtime.

💰

Usage limits

Hard caps on total API calls and tokens with --max-calls and --max-tokens. Never get a surprise bill.

🔌

OpenAI & Anthropic

Use any OpenAI or Anthropic model as the target or judge. Mix providers—run attacks against GPT-4o judged by Claude.

📋

JSON reports

Every test run outputs a structured JSON report with verdicts, severities, judge explanations, and raw model responses.

🔍

Manual review mode

Step through unclear and partial results interactively with pit review report.json and apply your own verdicts.

🚀

CI-friendly flags

Every option is available as a CLI flag. Drop PIT into any pipeline—GitHub Actions, GitLab CI, or your own scripts.

Get started

Up and running in two minutes

Requires Python 3.11+ and an OpenAI or Anthropic API key.

  • 1

    Clone & install

    Clone the repo, create a virtual env, and install PIT as an editable package.

  • 2

    Run the wizard

    Run pit run. You'll be guided through provider, model, system prompt, and attack selection.

  • 3

    Review the report

    Results print in real-time. A JSON report with all verdicts is saved automatically.

  • 4

    Harden & repeat

    Fix vulnerabilities, tighten your system prompt, and re-run until everything is green.
